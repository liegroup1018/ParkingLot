## 1. Motivation

The Parking Lot Management System requires three distinct user types: Super Users, Business Administrators, and Booth Attendants. Because this is a secure, internal, online-only facility application, there is no public self-service registration.

According to `prd.md` (§3.5), the system must provide a **"Dedicated interface for Admins to manage (Create, Read, Update, Delete) Attendant user accounts."**
While the backend API for listing and creating users (`GET/POST /api/v1/auth/users/`) was fully functional and documented, the frontend interface for this operation was missing from the Admin Dashboard.

This track introduces the "Staff & User Management" interface directly into the Admin Dashboard (`dashboard.html`), allowing existing Business Administrators to provision new Booth Attendants (or delegate access to new Administrators) entirely from the web UI.

---

## 2. Files Changed

### `apps/admin_ui/templates/admin_ui/dashboard.html`

- **Staff & User Management Section Added:**
  - Placed a new responsive data table below the Peak Hours chart.
  - Formatted the table to display User ID, Username, Email, Role (with distinct color badges), 2FA Setup Status, and Joined Date.
- **Create User Modal Added:**
  - Introduced a hidden `div#userModal` containing a form for new user provisioning.
  - Fields included: Username, Email, First Name, Last Name, Role dropdown (Admin / Attendant), Password, and Password Confirmation.
- **JavaScript Enhancements:**
  - `fetchUsers()`: Added logic to call `GET /api/v1/auth/users/` on page load.
  - `updateUsersTable()`: Added DOM manipulation to build table rows dynamically from the API JSON response.
  - `openUserModal()` & `closeUserModal()`: Logic for toggling modal visibility and resetting the form.
  - **Form Submit Handler:** Validates that passwords match client-side, then dispatches `POST /api/v1/auth/users/`. Upon a `201 Created` response, it closes the modal, clears the form, displays a success alert, and instantly refreshes the user table.

---

## 3. Behavioral Result

After this track:
1. **Business Administrators** can view a real-time list of all staff accounts currently registered in the system upon logging into the Admin Dashboard.
2. They can click "Create New User" to open a provisioning modal.
3. Upon successfully filling out the form, a new user is created in the backend, and an `AuditLog` entry (`USER_CREATED`) is automatically generated by the pre-existing API logic.
4. The newly created **Booth Attendants** can immediately log into the Attendant UI using the provisioned credentials.
